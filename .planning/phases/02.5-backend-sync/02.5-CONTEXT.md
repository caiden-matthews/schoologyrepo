# Phase 2.5: Backend Sync - Context

**Created:** 2026-05-05
**Status:** Ready for planning
**Type:** Gap Closure (custom event persistence across devices)

## Phase Boundary

Enable custom events and changes to sync across devices and browsers via Supabase backend database. Phase 2 handles local persistence (localStorage); Phase 2.5 adds server-side persistence for multi-device sync.

**In scope:**
- Supabase project setup and authentication
- Database schema for custom events
- API endpoints in Netlify Functions (POST/GET/DELETE custom events)
- Frontend sync logic for custom events
- Cross-device testing (web + mobile browsers)

**Out of scope:**
- Real-time sync (Phase 4+ enhancement)
- Offline-first sync conflict resolution
- Advanced data versioning

## Problem Statement

Current architecture stores custom events only in localStorage:
- ✅ Same browser persists across reloads
- ❌ Different browser/device loses all custom events
- ❌ Incognito/private mode loses events on close
- ❌ Multi-device users see different data per device

This breaks user expectations for cloud-based apps and is a soft launch blocker.

## Solution Overview

Use Supabase (PostgreSQL + Auth) to:
1. Store custom events server-side, keyed by user (OAuth Schoology token)
2. Sync custom events from frontend to backend on create/update/delete
3. Restore custom events on page load from backend
4. Keep localStorage as read-through cache (offline fallback)

## Technical Decisions

- **Database:** Supabase PostgreSQL (managed, auto-scaling, REST API)
- **Auth:** Schoology OAuth tokens (no separate user accounts needed)
- **Schema:** Simple `custom_events` table (id, user_id, event_data, created_at, updated_at)
- **Sync:** HTTP API in Netlify Functions (POST create, GET list, DELETE remove)
- **Conflict resolution:** Last-write-wins (simple, sufficient for soft launch)

## Implementation Tasks

1. Create Supabase project + database
2. Define custom_events table schema
3. Create Netlify Function endpoints (create, list, delete)
4. Add frontend sync logic (POST on create/delete, GET on page load)
5. Test cross-device sync (desktop + mobile, different browsers)
6. Update docs (Phase 4 real-time sync planning)

## Success Criteria

1. Custom events created in Browser A visible in Browser B after refresh
2. Custom events persist across device logout/login
3. Mobile browser can create/view/delete custom events with sync
4. No data loss on page reload
5. Sync completes within 2 seconds
6. Error handling: failed sync shows user-friendly message
