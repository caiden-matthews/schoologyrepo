# Schoology Study Planner — Public SaaS

## What This Is

A free, ad-supported web app that lets any student at any Schoology-powered school connect their account via OAuth and get a better academic dashboard than existing tools. Built on top of a working personal study planner (today view, week view, plan, grades, flashcards), this becomes a publicly launchable service similar to gunn.one and bessy — but with more features.

## Core Value

Any Schoology student can connect their account and immediately see a better, more feature-rich view of their academic life than they get from Schoology itself.

## Context

- **Existing codebase:** A fully functional single-HTML-file SPA (`study.html`, ~3056 lines) backed by Netlify serverless functions. The OAuth 1.0a flow is already implemented and working for one school. The main conversion work is making it generic and public-safe.
- **Current auth gap:** Existing Schoology consumer key/secret were obtained for personal use (possibly school-admin-issued). A Schoology Platform developer registration is required for any-school support — this is a prerequisite for launch.
- **Data architecture decision:** Using session-only storage (localStorage) means no database, no GDPR surface area, no user account system. The tradeoff is data loss if the user clears their browser, which is acceptable for v1.
- **Comparable products:** gunn.one and bessy are similar Schoology-adjacent tools. Differentiation is breadth of features (flashcards, custom events, richer views).
- **Monetization:** Google AdSense. Will need 6 months of traffic history for some AdSense policies — can apply for AdSense early and leave placeholders.

## Constraints

- **Tech stack**: Vanilla JS SPA + Netlify Functions — no bundler, no framework. Do not introduce npm or a build step.
- **Timeline**: Launch in a few weeks — scope must stay tight.
- **Schoology API**: Platform developer app registration required before any-school OAuth works. This is a dependency outside Claude's control.
- **Security**: Consumer key/secret must never appear in client-side code. Netlify env vars only.
- **No database**: v1 is session-only. Any feature requiring server-side user storage is deferred.
